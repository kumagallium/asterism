# Verification report ‚Äî Tier 0 function library is "enough"

**Date:** 2026-06-11 **Verdict:** ‚úÖ **sufficient** ‚Äî gate PASS. Originally `‚Ä¶Raw`
11.1% < 15%; after the nested-array exploder (`tabularize`) landed the corpus is
**0.0% < 5%** ‚Äî see the [Addendum](#addendum-2026-06-11--nested-arrays-closed-gate-15--5).
**Method/decision of record:** [`architecture/tier0-coverage-gate.md`](../architecture/tier0-coverage-gate.md) ¬∑
[`experiments/coverage-corpus/`](../../experiments/coverage-corpus/) ¬∑
harness `asterism_step0.coverage`

## 1. Question

Asterism ingests data through a **declarative path** (RML ‚Üí Morph-KGC) that may
reference only a **closed, once-vetted set of functions** (`asterism.functions`,
Tier 0) ‚Äî never generated code (`ingestion-execution-safety.md`). The design bet
is that a *small* head of functions, plus parameterized primitives, is **enough**
to onboard *arbitrary* datasets without the long tail forcing new code each time.

> **Is the Tier 0 head actually enough for arbitrary (non-starrydata) datasets,
> or does onboarding degrade into constant `‚Ä¶Raw` fallbacks?**

This needed to be a *measurement*, not an opinion ‚Äî the whole no-codegen safety
argument rests on the closed set staying small **and** sufficient.

## 2. Method

A deterministic harness (Track C) measures, over a diverse corpus, how often a
schema proposal's RML mapping has to fall back to a raw literal because no vetted
function fit.

- **Corpus** ‚Äî 14 cross-domain, **non-materials**, license-clean public datasets
  (weather, cars, film, finance, demography, ecology, disasters, labor, climate,
  aviation, seismology, bibliography √ó2, software). Sources + licences:
  [`experiments/coverage-corpus/SOURCES.md`](../../experiments/coverage-corpus/SOURCES.md);
  fetched + downsampled reproducibly by `fetch_corpus.py`.
- **Pipeline** ‚Äî for each dataset: `inspect` ‚Üí `propose` (a schema proposal whose
  ¬ß9 RML mapping is the unit of measurement) ‚Üí `materialize` extracts the RML ‚Üí
  the analyzer parses it. Proposals were generated by the Claude Code
  subscription agent **acting as the real `propose.SYSTEM_PROMPT`** (no Anthropic
  API billing), each column mapped faithfully to its real sample shape ‚Äî and are
  committed, so the report reproduces with no model call.
- **Metric & gate** ‚Äî of the columns a proposal treats as needing real
  computation (a vetted function **or** a `‚Ä¶Raw` fallback), the fraction that
  fell back to raw. **Gate: `‚Ä¶Raw` rate < `RAW_RATE_GATE = 0.15`.** Decision &
  rationale in the ADR.
- **Stability** ‚Äî the harness depends only on the project's stable interfaces
  (`inspect`, `materialize`, `rml_check.load_registry_fn_iris`), never on
  `functions.py` internals or the prompt text, so the same instrument measured
  the library at every stage of its growth.

## 3. Result

The gate was re-run at each Tier 0 milestone (same harness, same corpus method):

| Tier 0 state | functions | corpus `‚Ä¶Raw` rate | gate |
|---|--:|--:|:--:|
| v0 (original, starrydata-shaped) | 8 | **63.6%** | ‚ùå FAIL |
| + Track A (core) + Track B (primitives) | 25 | **36.8%** | ‚ùå FAIL |
| + multi-value (`split`/`array_at`/`json_array_single`) + corpus hardening | 28 | **11.1%** | ‚úÖ **PASS** |

Current run (28 functions, 14 datasets): **27 computed columns, 3 `‚Ä¶Raw`**.

- **Function usage is broad and earned** ‚Äî `array_at` √ó5, `date_iso` √ó3,
  `url_canonical` √ó3, `split` √ó3, `lookup` √ó2, `json_array_single` √ó2,
  `datetime_iso` √ó2, `bool_norm` √ó2, `doi_norm` √ó1, `year_only` √ó1. The
  starrydata-specific `qudt_*` / `float_array_*` stay unused on generic data, as
  expected ‚Äî confirming the head generalizes while the tail stays specialized.
- **The 3 remaining `‚Ä¶Raw` are genuinely irreducible** ‚Äî `crossref-works.author`
  (array of objects), `openlibrary-books.subject` and `github-repos.topics`
  (multi-element lists where *every* element is wanted). Each needs an RML
  **nested TriplesMap / iterator**, not a Tier 0 scalar function. They are the
  fallback working *as designed*, not a missing function.
- **T9 misses: none** ‚Äî every referenced function is in the closed set; the
  no-codegen guard-rail held across all proposals.
- **Demand by category** ‚Äî `messy_date`, `epoch_millis`, `url`, `doi`, `boolean`,
  and most `multivalue_or_json` columns are now handled by a function. The only
  category still mapped direct is `value_with_unit_name` (units in the *column
  name* with a clean numeric cell), where `value_of`/`unit_of` correctly do not
  apply ‚Äî modeling, not a gap.

## 4. Conclusion

**The Tier 0 head is empirically sufficient for arbitrary datasets.** Across a
14-dataset, non-materials corpus the proposal falls back to a raw literal on only
11.1% of computation-needing columns, and every one of those is an irreducible
nested/object array that *should* defer to RML iteration. The closed set stayed
small (28 functions) **and** became sufficient ‚Äî the no-codegen safety argument
holds without a long-tail escape hatch into generated code.

The gate stays at **15%** (not tightened to 10%): the remaining raws are
irreducible by design, so demanding 0% would penalize the correct use of the
fallback rather than signal a missing function. The open follow-up is the
*nested-array exploder* (RML nested TriplesMap / iterator) ‚Äî a separate
work-stream, not a Tier 0 function.

## 5. Limitations

- **Proposals are agent-authored, not API-sampled.** They follow the real
  system prompt and real per-dataset inspection (representative), but a different
  model/temperature varies at the margins. The `AnthropicLLMClient` path is
  available to cross-check.
- **The corpus is curated** to span domains and include computation-needing
  columns; the absolute rate reflects that mix.
- **Clean public data rarely ships *in-cell* messy scalars** (`$1,234`, `12%`,
  `10‚Äì20`, `300 K`), so `number_clean` / `percent_to_ratio` / `range_*` /
  `value_of` / `unit_of` get little exercise *from this corpus*; they target
  scraped / hand-maintained onboarding and are covered by the ingest package's
  own unit tests. Adding a scraped dataset would exercise them here.

## Addendum (2026-06-11) ‚Äî nested arrays closed, gate 15% ‚Üí 5%

The original report (¬ß3‚Äì4) called the three residual `‚Ä¶Raw` columns
(`crossref-works.author`, `openlibrary-books.subject`, `github-repos.topics`)
**irreducible** and left the gate at 15% because demanding 0% "would penalize the
correct use of the fallback." That premise no longer holds: the *nested-array
exploder* it named as the open follow-up has landed.

- **What changed** ‚Äî JSON sources are now **tabularized to CSV at ingest**
  (nested objects ‚Üí dot-path columns, arrays ‚Üí JSON-string cells), so the existing
  vetted Tier 0 exploders read them: `fn:json_pluck` for the object array (author),
  `fn:json_array` for the scalar arrays (subject, topics). No new function ‚Äî the T9
  closed set is unchanged. Decision of record:
  [`architecture/native-json-denormalization.md`](../architecture/native-json-denormalization.md).
- **Re-measurement** ‚Äî the three proposals were regenerated under the updated
  `propose` ¬ß9 (which now emits `ql:CSV` + the exploders for JSON sources, not a raw
  fallback). Same harness, same 14-dataset corpus:

  | Tier 0 + ingest state | corpus `‚Ä¶Raw` rate | gate |
  |---|--:|:--:|
  | 30 functions, native-JSON arrays irreducible | 11.1% | 15% ‚úÖ |
  | 30 functions, **+ JSON‚ÜíCSV tabularize** | **0.0%** | **5% ‚úÖ** |

  Current run: **27 computed columns, 0 `‚Ä¶Raw`**. `fn:json_array` √ó2,
  `fn:json_pluck` √ó1 now earn their place; `multivalue_or_json` demand (10 columns)
  is **100% function-handled, 0 raw**. T9 misses: none.
- **Gate tightened to `DEFAULT_RAW_RATE_GATE = 0.05`** ‚Äî with no irreducible raw
  left in the corpus, the loose-gate rationale is gone. 5% keeps real pressure
  (tolerates one genuinely novel irreducible structure ‚Äî array-of-arrays, a
  correlated child entity ‚Äî but trips on a regression) without demanding a brittle 0%.
- **Conclusion strengthened** ‚Äî Tier 0 + the JSON-tabularize boundary is sufficient
  for arbitrary datasets at **0.0%** raw across the corpus; the no-codegen safety
  argument holds with *no* long-tail escape, not merely a small one.
