ASTERISM · 引用できる事実レイヤー

散らばったデータを、
繋いで引用できる事実に。

Asterism は、散らばった構造化データ（CSV・JSON・API）を、信頼でき・ 引用でき・再現できる RDF に変換し、共有オントロジーで繋いで、 SPARQL と MCP で AI からも問える共有基盤です。スキーマ設計や自由質問では AI（LLM）も使いますが、事実の層は決定論的で、すべての値が出典まで辿れます。

このページは Asterism の核を、Starrydata の熱電材料データの小さなサブセットに 限定して体験できるデモです。このデモ自体はサーバも AI も使わず、 ブラウザ内の SPARQL エンジン（oxigraph-wasm）だけで動きます。

エンジンを初期化中…
Asterism の流れ
  1. 入力構造化データ（CSV / JSON / API）
  2. 変換宣言的に RDF 化＋共有オントロジーで接続
  3. 蓄積型付きの関係＋来歴(PROV)を持つグラフ
  4. 問うSPARQL / MCP（AI からも）
  5. 答え引用できる IRI・来歴・使った SPARQL を開示

ふつうのデータベースとの違い — データが「繋がって」いる

Asterism はデータを表のセルではなく、型付きの関係と 来歴（PROV）を持つ グラフとして扱います。そのため、このデモでは次の 3 つが実際に確かめられます。

来歴で引用できる どの数値も curve → sample → paper → digitization → ingestion と辿り、原典（DOI）まで来歴を示せる。→ 下の「来歴トレース」
IRI で一意に参照 各事実がグローバルに一意な IRI を持ち、そのまま引用・参照に使える。 → 各カードの引用 IRI
作り直さずに横断結合 同じ共有オントロジーを使う別データベース（Materials Project）と、データを統合用に 移し替え・作り直しせず同じ語彙のまま結合できる。→ 下の「横断結合」カード

ZT が最も高い熱電材料 property_ranking
