How to see the data behind it

The identifier works in SPARQL against the store that holds the dataset — your Asterism instance's 高度な検索 / SPARQL view, the API's /api/sparql, or the in-browser demo for the published subset:

DESCRIBE <…>

Answers produced by Asterism's verified tools cite these IRIs so the exact evidence row can be re-fetched, compared, and audited — identifier stability is the point; HTTP resolution is a convenience layered on top.

Why it doesn't resolve (yet)

Instance IRIs are currently identifiers only (see the namespace note): serving a description per IRI needs a live SPARQL service behind this domain, which a static GitHub Pages site cannot provide. A content-negotiated DESCRIBE responder (Turtle for machines, HTML for humans) is the planned phase 2.

If you followed a citation link and expected data, the citing Asterism instance is where the data lives — open the provenance panel there.
